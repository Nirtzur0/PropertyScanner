# CI

## Current State

A GitHub Actions workflow now exists at:
- `.github/workflows/ci.yml`

It runs two jobs:
- `docs-sync-guardrail`
- `offline-quality-gates`

## Command Mapping (Pointer-Based)

CI jobs reference command IDs from `docs/manifest/09_runbook.md`.

Current mapping:
- `docs-sync-guardrail` -> `CMD-DOCS-SYNC-GUARD`
- `docs-sync-guardrail` -> `CMD-CI-COMMAND-MAP-CHECK`
- `docs-sync-guardrail` -> `CMD-ARTIFACT-FEATURE-CONTRACT-CHECK`
- `offline-quality-gates` -> `CMD-CLI-HELP`
- `offline-quality-gates` -> `CMD-TEST-MARKERS`
- `offline-quality-gates` -> `CMD-TEST-UNIT`
- `offline-quality-gates` -> `CMD-TEST-INTEGRATION`
- `offline-quality-gates` -> `CMD-TEST-E2E`

## Required Checks Policy

Required checks:
- `docs-sync-guardrail`
- `offline-quality-gates`

Inside `offline-quality-gates`, required test stages are:
- deterministic unit suite
- offline integration suite
- offline e2e suite

Optional/non-blocking:
- live network/browser suite (`live` marker)

## Docs-System Integrity Guardrail

`docs-sync-guardrail` enforces docs updates when PR/push changes runtime/test/CI/config surfaces.

If changed files include `src/`, `tests/`, `.github/workflows/`, `config/`, or key dependency/test files, the change must also include:
- `docs/implementation/00_status.md`
- `docs/implementation/checklists/02_milestones.md`
- at least one manifest update among:
  - `docs/manifest/07_observability.md`
  - `docs/manifest/09_runbook.md`
  - `docs/manifest/10_testing.md`
  - `docs/manifest/11_ci.md`

Integrity guard:
- `scripts/check_command_map.py` fails if `docs/manifest/11_ci.md` references `CMD-*` IDs missing from `docs/manifest/09_runbook.md`.
- `scripts/check_artifact_feature_contract.py` fails if load-bearing artifact IDs are not mapped in alignment report/checklist/milestone surfaces.
